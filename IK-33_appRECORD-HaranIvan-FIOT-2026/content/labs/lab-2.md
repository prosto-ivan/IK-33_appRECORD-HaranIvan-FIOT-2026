## 🗄 1-2. Нормалізація та створення структури БД

Для забезпечення цілісності даних у проєкті **LinkEdu Hub** було впроваджено нормалізовану структуру. Замість простого збереження тексту, ролі та типи контенту винесені в окремі таблиці-довідники:

1.  **Roles:** (`role_id`, `role_name`) — адмін, користувач.
2.  **ResourceTypes:** (`type_id`, `type_name`) — відео, стаття, курс.
3.  **Users:** (`user_id`, `username`, `email`, `password_hash`, `role_id`).
4.  **Resources:** (`resource_id`, `title`, `url`, `type_id`, `created_by`).

## ⚙️ 4-6. Підключення Node.js та використання Sequelize

Для взаємодії з MS SQL Server використано **Sequelize ORM** та драйвер `tedious`. Конфігурація підключення (`database.js`) забезпечує автоматичну перевірку з'єднання при старті сервера.

Використання ORM дозволило:
- Замінити ручне написання SQL на методи `findAll`, `create`, `update`.
- Автоматизувати створення та оновлення таблиць через синхронізацію моделей.

## 📂 7-8. Моделі та зв'язок One-to-Many

Було створено моделі `User` та `Resource`. Ключовим етапом стало налаштування зв'язку **Один-до-Багатьох**: один користувач (адмін) може створювати багато навчальних матеріалів.

**Код асоціації у `server.js`:**
```javascript
User.hasMany(Resource, { foreignKey: 'created_by', as: 'createdResources' });
Resource.belongsTo(User, { foreignKey: 'created_by', as: 'creator' });
```
При виконанні запиту SELECT, Sequelize автоматично генерує LEFT OUTER JOIN, що дозволяє отримати автора ресурсу одним запитом до БД.

## 🚀 5. Результати виконання операцій (CRUD)
Запуск скрипта server.js продемонстрував повний цикл роботи з даними:

- SELECT: Пошук користувача разом із його списком ресурсів.

- UPDATE: Корекція заголовка ресурсу в базі даних.
